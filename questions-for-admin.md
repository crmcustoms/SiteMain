# Вопросы для администратора Docker/сервера

## Цель
Настроить автоматическое развертывание сайта: когда код изменяется в GitHub, сайт должен автоматически обновиться в Docker.

## Текущая ситуация
- Сайт уже работает в Docker контейнере
- Код проекта находится локально и планируется загрузить в GitHub
- Нужно связать GitHub → Docker для автоматических обновлений

## Команды для получения информации

Если можете, выполните эти команды и пришлите результат:

```bash
# Список запущенных контейнеров
docker ps

# Подробная информация о контейнере (замените CONTAINER_NAME)
docker inspect CONTAINER_NAME

# Если есть docker-compose
docker-compose ps

# Проверка доступности Git
git --version

# Проверка места на диске
df -h
```

## Предпочтения по развертыванию

Какой подход предпочитаете:

**Вариант А: GitHub Webhooks**
- GitHub отправляет уведомление на сервер при изменении кода
- Сервер автоматически обновляет контейнер
- Требует открытый endpoint на сервере

**Вариант Б: GitHub Actions**
- GitHub Actions собирает и отправляет новый образ
- Может использовать Docker Hub или собственный registry
- Не требует открытых портов на сервере

**Вариант В: Polling (опрос)**
- Сервер периодически проверяет изменения в GitHub
- Простой в настройке
- Менее оперативный
