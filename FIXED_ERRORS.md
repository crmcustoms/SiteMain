# Исправление ошибок сборки

В проекте были исправлены следующие ошибки, которые приводили к проблемам при сборке:

## 1. Неправильный импорт компонента Footer

В файле `app/[lang]/landing/layout.tsx` был неправильный импорт компонента Footer:

```typescript
// Было:
import { Footer } from "@/components/footer"

// Стало:
import Footer from "@/components/footer"
```

Проблема заключалась в том, что компонент Footer экспортируется как экспорт по умолчанию (default export), но импортировался как именованный экспорт.

## 2. Неправильная передача словаря в компонент ContactFormDialog

В файле `app/[lang]/landing/audit-crm/page.tsx` словарь передавался некорректно:

```typescript
// Было:
dict={dict.common}

// Стало:
dict={dict}
```

Компонент ContactFormDialog ожидает полный словарь, а не только его часть.

## Инструкции по деплою

1. Убедитесь, что все изменения сохранены в проекте
2. Соберите проект локально для проверки: `npm run build`
3. Если сборка прошла успешно, загрузите проект на сервер
4. На сервере выполните сборку Docker-образа: `docker build -t sitemain .`
5. Запустите контейнер: `docker run -p 3000:3000 sitemain`

## Дополнительные рекомендации

1. При разработке новых компонентов и страниц обращайте внимание на правильное использование импортов и экспортов
2. Проверяйте передачу правильных пропсов в компоненты
3. Перед деплоем всегда выполняйте локальную сборку для проверки наличия ошибок
