# Вопросы по архитектуре проекта

## Технические вопросы

1. **Клиентская часть:**
   - Какой подход к управлению состоянием предпочтителен? Нужно ли использовать глобальное хранилище (Redux, Context API) или достаточно локального состояния компонентов?
   - Какие анимации планируются использоваться на сайте? Есть ли предпочтения по библиотекам анимаций (Framer Motion, React Spring)?
   - Требуется ли реализация серверного рендеринга для всех страниц или только для критичных с точки зрения SEO?

2. **Интеграции:**
   - Какие конкретно данные будут храниться в Notion и каким образом их нужно отображать на сайте?
   - Нужна ли интеграция с CRM-системами для обработки заявок от клиентов?
   - Требуется ли реализация онлайн-чата для общения с потенциальными клиентами?

3. **Безопасность:**
   - Какие требования по безопасности предъявляются к формам и передаче данных пользователей?
   - Нужна ли защита от DDoS-атак и других видов угроз?
   - Требуется ли шифрование данных клиентов при отправке форм?

4. **Производительность:**
   - Какие целевые показатели Core Web Vitals необходимо достичь?
   - Есть ли особые требования к производительности на мобильных устройствах?
   - Нужна ли оптимизация для пользователей с медленным интернетом?

5. **Масштабирование:**
   - Планируется ли значительное расширение функциональности в будущем?
   - Нужно ли закладывать возможность масштабирования для большого объема трафика?
   - Как часто планируется обновление контента и функций сайта?

## Вопросы по функциональности

1. **Мультиязычность:**
   - Какие языки должен поддерживать сайт? Есть ли приоритетные языки?
   - Нужно ли локализовать не только текст, но и другие элементы (даты, валюту, форматы чисел)?
   - Как должно происходить определение языка пользователя (автоматически, по выбору)?

2. **Услуги автоматизации:**
   - Какие конкретные услуги автоматизации будут представлены на сайте?
   - Нужна ли система фильтрации или каталогизации услуг?
   - Требуется ли расчет стоимости услуг онлайн?

3. **Кейсы и портфолио:**
   - Насколько детально должны быть представлены кейсы?
   - Нужна ли система фильтрации или поиска среди кейсов?
   - Требуется ли возможность загрузки подробных материалов по кейсам?

4. **Взаимодействие с пользователями:**
   - Какие данные необходимо собирать через формы обратной связи?
   - Нужна ли система аутентификации для клиентов?
   - Требуется ли функционал для бронирования консультаций онлайн?

5. **Контент:**
   - Как часто будет обновляться контент на сайте?
   - Нужен ли функционал для автоматической публикации материалов из Notion?
   - Требуется ли система комментариев для блога?

## Вопросы по дизайну и UX

1. **Визуальный стиль:**
   - Есть ли фирменный стиль, которого нужно придерживаться?
   - Какие цветовые схемы предпочтительны?
   - Нужна ли поддержка темной темы?

2. **UX/UI:**
   - Какие особенности целевой аудитории следует учитывать при разработке интерфейса?
   - Есть ли особые требования к навигации и структуре сайта?
   - Нужны ли дополнительные элементы взаимодействия (интерактивные графики, 3D-элементы)?

3. **Доступность:**
   - Какие требования по доступности (a11y) необходимо соблюдать?
   - Нужна ли поддержка скринридеров и других вспомогательных технологий?
   - Есть ли особые требования к контрастности и размеру шрифтов?
