# Руководство по замене изображений на сайте

Этот документ содержит информацию о всех местах, где необходимо заменить изображения на сайте для улучшения визуального представления.

## Структура папок для изображений

Для организации изображений создана следующая структура папок:

```
public/
  ├── images/
  │   ├── landings/
  │   │   ├── audit/         # Изображения для страницы аудита CRM
  │   │   ├── implementation/# Изображения для страницы внедрения CRM
  │   │   ├── support/       # Изображения для страницы поддержки CRM
  │   │   ├── development/   # Изображения для страницы индивидуальной разработки
  │   │   │   └── icons/     # Иконки технологий для страницы разработки
  │   │   ├── project-management/ # Изображения для страницы управления проектами
  │   │   ├── industry-solutions/ # Изображения для страницы отраслевых решений
  │   │   └── common/        # Общие изображения для всех лендингов
  │   ├── hero/              # Изображения для героя на главной странице
  │   ├── industries/        # Изображения для секции отраслей
  │   ├── cases/             # Изображения для кейсов
  │   └── testimonials/      # Фото для отзывов
  ├── img/                   # Общие изображения и иконки
  └── favicons/              # Фавиконы
```

## Рекомендации по подготовке изображений

1. **Оптимизация размера:** Используйте сервисы оптимизации изображений (например, TinyPNG, Squoosh) для уменьшения размера файлов без потери качества.
2. **Формат файлов:** 
   - Для фотографий: JPEG или WebP
   - Для иконок и изображений с прозрачностью: PNG или SVG
   - Для анимаций: GIF или WebP
3. **Адаптивность:** Подготовьте несколько версий каждого изображения для разных разрешений экрана.
4. **Alt-текст:** При замене изображений не забудьте обновить alt-текст для улучшения SEO и доступности.
5. **Брендирование:** По возможности используйте изображения, соответствующие фирменному стилю (цвета, шрифты, логотип).

## Процесс замены

1. Подготовьте все необходимые изображения согласно указанным размерам и форматам.
2. Разместите новые изображения в соответствующих директориях (`/public/images/`, `/public/img/`, `/public/favicons/`).
3. Обновите пути к изображениям в указанных файлах компонентов.
4. Проверьте отображение изображений на разных устройствах и в разных браузерах.
5. Убедитесь, что все изображения корректно загружаются и отображаются.
